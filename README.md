## 🚀 Local Setup

### Prerequisites
- Python 3.10 or 3.11
- pip
- Internet connection (for yfinance)

### Step 1 — Clone and install

```bash
git clone https://github.com/YOUR_GITHUB_USERNAME/stock-dashboard.git
cd stock-dashboard
pip install -r requirements.txt
```

> **Note:** `sqlite3` is part of the Python standard library — it is not in `requirements.txt` and does not need a separate install.

### Step 2 — Build the database

**Run this from the project root:**

```bash
python backend/data.py
```

This fetches 1 year of NSE data, cleans it, computes metrics, and saves:
- `data/stock_data.csv`
- `backend/database.db` (tables: `stock_data` + `stock_snapshot`)

Expected output:
```
🚀 Stock Data Pipeline
✅ RELIANCE: 248 rows
✅ TCS: 248 rows
...
✅ Pipeline complete! 10 stocks | 2,480 rows
```

### Step 3 — Start the API

**Run from the `backend/` folder:**

```bash
cd backend
uvicorn main:app --reload
```

API live at: **http://localhost:8000**
Swagger docs: **http://localhost:8000/docs**

### Step 4 — Open the dashboard

Open `frontend/index.html` in your browser,
or visit **http://localhost:8000/dashboard**.

## 📊 Calculated Metrics

All percentage fields follow a consistent convention throughout the codebase, database, and API:
> **A value of `+1.25` means the stock moved +1.25%. A value of `-0.80` means -0.80%.**

| Column              | Formula                                            | Unit |
|---------------------|----------------------------------------------------|------|
| `daily_return`      | `(close − open) / open × 100`                     | %    |
| `ma_7`              | Rolling 7-day mean of close                        | ₹    |
| `ma_30`             | Rolling 30-day mean of close                       | ₹    |
| `high_52w`          | Rolling max of close over 252 trading days         | ₹    |
| `low_52w`           | Rolling min of close over 252 trading days         | ₹    |
| `volatility_30d`    | Rolling 30-day std dev of `daily_return`           | %    |
| `price_momentum`    | `(close − close[30d ago]) / close[30d ago] × 100` | %    |

**Safety guards in `data.py`:**
- Dates use `pd.to_datetime(..., errors="coerce")` — bad dates become NaT and are dropped
- `open == 0` is replaced with NaN before computing `daily_return` (no divide-by-zero)
- `sqlite3` context manager (`with connect(...) as conn`) — no manual close needed

## 🤖 ML Models

### Linear Regression (`/stocks/{symbol}/predict`)
- **Features:** integer day index
- **Target:** daily close price
- **Training data:** last 90 trading days
- **Output:** R² score + 7/14/30-day price forecast

**Known limitations:**
- R² measures how well a straight line fits *past* data — it does not predict future accuracy
- Cannot capture volatility, earnings surprises, or macro events
- If R² < 0.5, the API returns a low-confidence warning
- Call it a "trend-based demo prediction" — not a real forecasting system

### LSTM Deep Learning (`/stocks/{symbol}/predict/lstm`)
- **Architecture:** `LSTM(64) → Dropout(0.2) → LSTM(32) → Dropout(0.2) → Dense(1)`
- **Look-back window:** 30 days
- **Training data:** last 120 days (80/20 train/test split)
- **Metrics:** MAE and RMSE on held-out test set
- **Note:** First request trains the model (~20–40 s)

### Sentiment Analysis (`/stocks/{symbol}/sentiment`)
- **Method:** VADER (Valence Aware Dictionary and sEntiment Reasoner)
- **Scale:** compound score from -1.0 (very negative) to +1.0 (very positive)
- **Signal:** BUY (≥ 0.15) / HOLD (-0.15 to 0.15) / SELL (≤ -0.15)
- **Data:** set `NEWSAPI_KEY` env var for live news; uses curated mock headlines otherwise

### Multi-Stock Ranking (`/market/predict-all`)
- Runs Linear Regression on all stocks simultaneously
- Ranks by predicted % change over N trading days
- Includes BUY/HOLD/SELL signal and R²-based confidence label per stock

## 🔗 Correlation Analysis

Pearson correlation of `daily_return` between two stocks:

| Range       | Meaning                              |
|-------------|--------------------------------------|
| ≥ 0.7       | Strong positive — move together      |
| 0.3 – 0.7   | Moderate positive                    |
| -0.3 – 0.3  | Little / no relationship             |
| -0.7 – -0.3 | Moderate negative                    |
| ≤ -0.7      | Strong negative — move opposite      |

Returns a friendly error if the same symbol is passed twice.

## 💡 Design Decisions

1. **SQLite over PostgreSQL** — zero config, built into Python, perfect for demo scale.
2. **`stock_snapshot` table** — one row per symbol for fast gainers/losers queries.
3. **yfinance** — free, no API key, real NSE data via `.NS` suffix.
4. **FastAPI** — auto-generates Swagger docs, modern Python async framework.
5. **Chart.js** — lightweight, no build step, works directly in HTML.
6. **VADER for sentiment** — rule-based NLP designed for short social/financial text; no GPU required.
7. **context managers** — `with sqlite3.connect(...) as conn` prevents connection leaks.
